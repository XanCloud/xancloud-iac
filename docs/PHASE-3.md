# Phase 3 — Escala o Pivote

## Objetivo

Decidir el siguiente paso según datos reales de uso: escalar (más módulos, más clientes, soporte) o pivotar (cambiar alcance, stack o modelo de entrega).

## Entradas esperadas

- Métricas de uso (entornos desplegados, incidencias, tiempo de onboarding).
- Feedback de clientes o equipos internos.
- Coste de mantenimiento vs. valor entregado.

## Opciones

- **Escalar**: Más blueprints, más clouds (si aplica), documentación pública, posible open-source controlado.
- **Pivote**: Reducir alcance, cambiar a otro tooling, o integrar en otro producto/servicio.

## Notas

- Esta fase no tiene entregables de código concretos; es decisión estratégica basada en Phase 1 y 2.
- DECISIONS.md y RISKS.md deben reflejar las conclusiones.
